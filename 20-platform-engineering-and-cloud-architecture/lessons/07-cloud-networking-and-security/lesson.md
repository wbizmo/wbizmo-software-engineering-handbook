# Lesson 07: Cloud Networking And Security

## Learning Objectives

- Understand cloud networking.
- Understand VPCs.
- Understand load balancers.
- Understand security boundaries.
- Understand zero-trust concepts.

---

## Introduction

Cloud platforms depend heavily on networking and security.

Without proper controls:

- services become exposed
- data becomes vulnerable
- outages become more likely

---

## Virtual Private Cloud (VPC)

Private network inside cloud environment.

Benefits:

- isolation
- segmentation
- security

---

## Subnets

Divide networks into smaller sections.

Examples:

- public subnet
- private subnet

---

## Load Balancers

Distribute traffic across services.

Benefits:

- scalability
- availability
- fault tolerance

---

## Security Groups

Firewall-like controls.

Define:

- allowed traffic
- blocked traffic

---

## Identity And Access Management

Controls permissions.

Examples:

- users
- roles
- services

Principle:

Least privilege.

---

## Secrets Management

Examples:

- API keys
- database credentials
- tokens

Never hardcode secrets.

---

## Encryption

Protect data:

- in transit
- at rest

---

## Zero Trust

Principle:

Trust nothing automatically.

Verify everything.

---

## Platform Engineering Connection

Platform teams often define:

- networking standards
- security controls
- access policies

for entire organizations.

---

## Summary

Cloud networking and security provide the foundation for safe and reliable platform operations.
