# Redis, Caching & Background Jobs

## Overview

Redis is one of the most widely used backend infrastructure technologies.

It is commonly used for:

- Caching
- Sessions
- Rate Limiting
- Job Queues
- Pub/Sub
- Distributed Locks
- Real-time Systems

This module teaches Redis fundamentals, caching strategies, queue processing, BullMQ, retries, scheduled jobs and production workflows.

## Why This Module Matters

Without Redis:

- Databases become overloaded
- APIs become slower
- Background processing becomes difficult

With Redis:

- Faster responses
- Reduced database load
- Better scalability
- Reliable job processing

## Learning Outcomes

By the end of this module you will be able to:

- Explain Redis
- Store data in Redis
- Implement caching
- Design cache strategies
- Build job queues
- Use BullMQ
- Handle retries
- Handle failures
- Schedule jobs
- Build worker systems
