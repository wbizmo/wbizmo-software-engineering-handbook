## What Is Version Control?

Version control is a system that tracks changes to files over time.

Think of it as:

- A save history for code.
- A timeline of project changes.
- A way to recover previous versions.
- A collaboration tool for teams.

Without version control:

- Mistakes are difficult to undo.
- Collaboration becomes risky.
- File duplication becomes messy.
- Project history disappears.

Example:

    project-final.zip
    project-final-final.zip
    project-final-final-v2.zip
    project-final-final-v2-fixed.zip

This becomes impossible to manage.

Version control solves this.

## Why Developers Use Version Control

Developers use version control to:

- Track changes
- Restore older versions
- Collaborate safely
- Review code
- Experiment without fear
- Maintain project history

Almost every professional software project uses version control.

## What Is Git?

Git is the most popular version control system in the world.

Created by:

    Linus Torvalds

The creator of Linux.

Git is:

- Fast
- Distributed
- Reliable
- Free
- Open source

Git powers millions of projects worldwide.

## What Is A Repository?

A repository (repo) is a project tracked by Git.

Examples:

    cashflowr
    syncgrid-api
    vaultbox-api
    pulseforms
    nommo

Each repository contains:

- Source code
- Documentation
- History
- Commits
- Branches

## Local Repository

A local repository exists on your computer.

Example:

    C:\Projects\cashflowr

or

    ~/workspace/cashflowr

You can work completely offline.

## Remote Repository

A remote repository exists on a service such as GitHub.

Examples:

- GitHub
- GitLab
- Bitbucket

Remote repositories help:

- Backup code
- Share code
- Collaborate

## What Is A Commit?

A commit is a snapshot of your project.

Think of it as:

    Save Point

Each commit should represent a meaningful change.

Examples:

    Add authentication system
    Fix login bug
    Create dashboard page
    Add API documentation

## Benefits Of Git

- Recover mistakes
- Safe experimentation
- Professional workflows
- Team collaboration
- Deployment integration
- CI/CD integration
